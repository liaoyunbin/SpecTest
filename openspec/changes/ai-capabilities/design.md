## 核心原则

```
┌─────────────────────────────────────────────────────────────────┐
│                  用好 AI 的核心原则                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 越具体越好                                                  │
│     ──────────                                                  │
│     ✗ "帮我写代码"                                              │
│     ✓ "帮我写一个 Unity C# 对象池，支持泛型，带预热功能"        │
│                                                                 │
│  2. 给上下文                                                    │
│     ────────                                                    │
│     ✗ "这个 bug 怎么修？"                                       │
│     ✓ "Unity 2022.3，UI 面板打开后按钮没响应，代码在..."        │
│                                                                 │
│  3. 分步来                                                      │
│     ──────                                                      │
│     大任务拆成多轮对话，每轮处理一个子问题                       │
│                                                                 │
│  4. 验证                                                         │
│     ──────                                                       │
│     AI 可能出错（幻觉），重要信息需查证                          │
│     生成的代码要 Review，不要盲信                               │
│                                                                 │
│  5. 迭代                                                         │
│     ──────                                                       │
│     第一版不满意就让它改，不要一次放弃                          │
│     "再换一种写法"、"能不能更简洁"、"加个错误处理"              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## AI 边界

```
┌─────────────────────────────────────────────────────────────────┐
│                    AI 做不到的事情                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ✗ 感知现实世界 — 看不到你的屏幕、听不到声音                     │
│  ✗ 操作软件 — 不能帮你点鼠标、拖拽 UI、运行 Unity               │
│  ✗ 100% 正确 — 会产生幻觉，生成不存在的方法名                    │
│  ✗ 知道最新信息 — 知识有截止日期（除非联网）                     │
│  ✗ 替代你的判断 — 最终决策权在你自己                            │
│                                                                 │
│  安全红线                                                        │
│  ──────────                                                      │
│  ✗ 不要输入密码、密钥、Token                                     │
│  ✗ 不要输入商业机密                                             │
│  ✗ 不要用于医疗诊断                                             │
│  ✗ 不要用于正式法律文件（仅供参考）                              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```
