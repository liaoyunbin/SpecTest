# 小说架构设计

## 整体架构

### 三卷式结构（30万字）

```
第一卷：建立秩序（1-10万字）
├── 把混乱避难所改造成有序小区
├── 核心爽点：秩序建立
└── 发展轨迹：10人 → 基础秩序

第二卷：完善系统（10-20万字）
├── 把有序小区升级为舒适家园
├── 核心爽点：生活质量提升
└── 发展轨迹：100人 → 舒适生活

第三卷：文明重建（20-30万字）
├── 把舒适家园建设成文明标杆
├── 核心爽点：文明传承
└── 发展轨迹：1000人 → 文明中心
```

## 主角设定

### 林晚人物档案

**基本信息**
- 姓名：林晚
- 年龄：28岁
- 职业：前地产项目经理
- 性格：极致精致生活强迫症 + 极致理性

**强迫症表现**
- 视觉强迫：无法忍受不协调的色彩和混乱的布局
- 听觉强迫：噪音超过60分贝就头痛
- 嗅觉强迫：对异味极度敏感
- 触觉强迫：追求材质光滑和温度适宜

**行为模式**
```
问题出现 → 觉得不爽 → 分析原因 → 制定方案 → 执行解决 → 获得爽感
```

**核心特质**
- 永不妥协的精致标准
- 极致理性的决策能力
- 强大的执行力和项目管理能力
- 对人情的冷漠和对效率的追求

## 章节架构

### 每章固定模板（确保爽点密度）

```markdown
1. 开头钩子（200字）：问题出现
2. 林晚反应（100字）：觉得不爽
3. 问题分析（300字）：强迫症视角
4. 方案制定（400字）：完美解决方案
5. 执行过程（500字）：改造实施
6. 成果展示（300字）：秩序建立
7. 林晚满意（100字）：获得爽感
8. 结尾钩子（100字）：新问题出现
```

### 爽点控制机制

| 爽点类型 | 出现频率 | 具体表现 |
|---------|---------|---------|
| 小爽点 | 每章 | 解决具体问题，建立小秩序 |
| 中爽点 | 每3章 | 完成重要建设，实现突破 |
| 大爽点 | 每10章 | 实现重大目标，阶段成就 |

## 世界观设定

### 末日背景

**天灾类型**：全球气候异常（持续3年）
- 极端天气频发
- 资源供应中断
- 社会秩序崩溃

**社会状态**
- 政府功能基本瘫痪
- 局部地区秩序重建
- 幸存者自发组织

**资源状况**
- 物资稀缺但可获取
- 技术设备部分保留
- 人力资源成为关键

### 特殊设定

**异能者出现**（作为工具人）
- 出现概率低，能力有限
- 主要用于解决特定问题
- 不改变末日基本设定

**文明方舟系统**（金手指）
- 高级工具箱+智能说明书
- 按需使用，不频繁出现
- 服务于秩序建设目标

## 写作技巧指南

### 保持人设一致性

1. **情绪控制**：林晚的情绪范围限定在"不爽"→"满意"
2. **行为逻辑**：永远选择成本效益比最高的方案
3. **标准坚持**：在可能范围内坚持精致生活标准
4. **理性决策**：不考虑人情，只考虑效率

### 制造强烈反差

1. **环境反差**：末日废墟 vs 精致小区
2. **行为反差**：别人将就 vs 林晚讲究
3. **结果反差**：混乱世界 vs 有序天堂
4. **价值观反差**：生存优先 vs 生活品质

### 控制节奏感

1. **快节奏**：问题出现 → 立即解决
2. **强反差**：混乱 → 秩序的快速转变
3. **高密度**：每章至少一个主要爽点
4. **预期管理**：让读者知道每章都有问题要解决

## 风险控制

### 合理性控制

1. **适度夸张**：在可能的情况下追求精致
2. **避免过度**：不写违背基本生存逻辑的情节
3. **保持真实**：让读者觉得"虽然夸张，但有可能"

### 爽点多样化

1. **秩序爽**：混乱变有序
2. **建设爽**：简陋变精美
3. **管理爽**：低效变高效
4. **成就爽**：小目标到大目标

这个架构确保了小说的爽文属性，同时保持了逻辑合理性，为30万字的创作提供了清晰的路线图。
