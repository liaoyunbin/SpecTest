## 主角设定

### 林晚人物档案

**身份设定**：
- **穿越者**：来自21世纪的重度强迫症患者，因加班猝死意外穿越
- **附身对象**：原主"林晚"，被规则污染吞噬的幸存者
- **记忆融合**：继承原主全部记忆，包括对规则污染的恐惧

**基本信息**
- 姓名：林晚（沿用原主姓名）
- 年龄：28岁
- 穿越前职业：前资深物业管理员（对规则、条例有极致追求）
- 性格：重度规则强迫症 + 极致理性 + 对逻辑矛盾有生理性不适

**强迫症表现**
- **文本强迫**：无法忍受格式错误、语病、逻辑矛盾的文字
- **排版强迫**：字体大小、标点符号、段落格式必须规范统一
- **逻辑强迫**：对矛盾、歧义、漏洞有超常洞察力和生理性不适
- **秩序强迫**：一切必须按规则运行，不可有例外

**核心能力**：【规则纠错】
- **看穿漏洞**：能直观看到诡异规则中的逻辑漏洞和矛盾之处
- **规则修正**：通过批注、修订规则文本，改写规则效力
- **规则转化**：将"绝对杀人规则"弱化为"限制性规则"或"可规避流程"
- **规则固化**：修正后的规则会固化生效，形成稳定的秩序领域

**行为模式**
```
规则出现 → 发现漏洞（生理不适）→ 分析逻辑 → 红笔修订 → 规则固化 → 获得爽感
```

**解决问题的独特方式**：
- **矛盾规则**：用修正案消除矛盾，如"修正案1：22点后，仅当确认门外为人类且发出明确求助信号时，方可开门"
- **杀人规则**：将其转化为"限制性规则"或"需履行前置义务"
- **混乱区域**：发布《安全区管理草案》，建立清晰有序的规则体系

**标志性行为**：
- 随身携带一支红色批注笔
- 对所有规则文本进行"格式审查"和"逻辑校对"
- 口头禅："这规则有漏洞！"、"格式不对，重写！"、"根据《管理条例》第X条..."

**与系统的互动方式**：
- 系统检测到规则污染 → 生成"规则修正"任务
- 林晚分析规则漏洞 → 用红笔批注修订
- 通过建立制度、制定规则、优化流程来从根本上解决
- 完成任务后获得双重爽感：规则固化 + 秩序建立

### 规则污染设定

**规则形式**：
- 以文字形式呈现（血字、刻痕、投影等）
- 格式混乱、逻辑矛盾、语病百出
- 普通人遵循或对抗这些规则九死一生

**污染等级**：
| 等级 | 表现 | 危险程度 |
|------|------|---------|
| 轻度 | 规则逻辑基本完整，仅有少量漏洞 | 可规避 |
| 中度 | 规则矛盾明显，存在致命陷阱 | 高风险 |
| 重度 | 规则完全混乱，现实扭曲严重 | 极度危险 |

**规则类型**：
- **限制规则**：禁止或限制某些行为（如"不可在22点后开门"）
- **强制规则**：强制要求执行某些行为（如"听到婴儿啼哭必须开门"）
- **惩罚规则**：违反规则的后果描述
- **环境规则**：改变环境状态的规则

**规则解析系统**（金手指）

**系统名称**：规则解析系统（简称"解析系统"）

**核心目的**：
- 解析诡异规则的逻辑结构和漏洞
- 辅助宿主进行规则修正和重构
- 记录已修正规则，建立秩序数据库

**核心原则**：
- **规则可视化**：将抽象规则转化为可分析的文本结构
- **漏洞标记**：自动标记规则中的逻辑矛盾和格式错误
- **知识辅助**：提供规则修正的理论支持

**系统来源**：
- 伴随穿越而来的高维文明规则解析装置
- 检测到宿主对规则异常的敏感性后自动激活
- 系统目的：协助宿主在规则污染世界中建立秩序

**绑定过程**：
- 穿越瞬间，系统与林晚的规则感知能力同步
- 灵魂绑定，与宿主的强迫症特质深度融合
- 绑定初期有3天适应期，解析能力逐步解锁
- 系统会持续扫描周围的规则污染，标记需要修正的规则

**规则检测机制**：
- **规则扫描**：实时扫描周围环境中的规则文本
- **漏洞识别**：自动识别逻辑矛盾、格式错误、语病
- **风险评估**：评估规则的危险程度和影响范围
- **优先级排序**：根据危险程度和宿主不适程度排序

**任务生成规则**：
- **触发条件**：检测到规则污染或宿主产生规则不适
- **任务描述**："修正XX区域的规则漏洞"、"规范XX规则文本"
- **奖励机制**：完成任务后提升规则解析能力或解锁新功能

**任务系统**：

| 任务类型 | 触发条件 | 奖励类型 |
|---------|---------|---------|
| **规则修正** | 检测到矛盾规则 | 规则解析等级提升 |
| **区域净化** | 发现污染区域 | 秩序领域扩展 |
| **异常收容** | 遭遇规则寄生体 | 收容物管理权限 |
| **规则制定** | 需要建立新规则 | 规则模板解锁 |

**核心功能**：

| 功能模块 | 描述 | 使用限制 |

- 宿主需亲自进行规则批注和修订
- 系统可提供规则模板和修正建议

## 写作技巧指南

### 保持人设一致性

1. **规则强迫症**：林晚对逻辑矛盾、格式错误有生理性不适
2. **行为逻辑**：永远从根源解决问题，通过规则修正而非暴力
3. **标准坚持**：规则必须规范、统一、可执行
4. **理性决策**：不考虑人情，只考虑规则合理性

### 制造强烈反差

1. **规则反差**：混乱矛盾的诡异规则 vs 规范有序的修正规则
2. **行为反差**：别人恐惧规则 vs 林晚吐槽规则漏洞
3. **结果反差**：必死规则 vs 可规避流程
4. **价值观反差**：原始混乱崇拜 vs 绝对秩序信仰

### 控制节奏感

1. **快节奏**：规则出现 → 发现漏洞 → 红笔修正 → 规则固化
2. **强反差**：混乱规则引发死亡 vs 修正规则建立安全区
3. **高密度**：每章至少一个规则修正爽点
4. **预期管理**：让读者期待林晚如何"批改"下一条规则

### 战斗方式设计

1. **面对不可直视之影**：递上《可视光谱登记表》："请先登记你的形态和可见波长，未登记实体不予对视。"
2. **面对必须回答的谜语**：退回谜面："问题表述存在歧义，请使用标准疑问句重新提问，并提供至少两个逻辑上互斥的选项。"
3. **面对杀人规则**：添加前置义务："根据《异常实体临时管理条例》第7条第3款，你的杀人流程缺少前置告知义务，本次攻击无效。"

### 标志性台词

1. "这规则有漏洞！"
2. "格式不对，重写！"
3. "根据《管理条例》第X条第Y款..."
4. "欢迎来到我的领域。在这里，恐怖也需要排队取号，并按流程表进行。"
5. "你的存在本身，就违反了基本物理法则第3章。这是整改通知书，请在24小时内提交《合理化存在解释报告》。"

## 风险控制

### 合理性控制

1. **规则逻辑**：修正后的规则必须逻辑自洽
2. **能力限制**：规则修正需要时间和能量，不能无限使用
3. **人性冲突**：绝对秩序可能引发人性异化，增加故事深度

### 爽点多样化

1. **规则修正爽**：从根源解决问题，消除矛盾规则
2. **秩序建立爽**：混乱区域变成安全区
3. **异常收容爽**：将危险诡异转化为可控资源
4. **规则制定爽**：发布新规则，建立秩序体系
5. **逻辑战争爽**：与混乱源头进行规则定义的终极对决

### 特色细节设计

1. **给会哭泣的雕像划定哭泣时间（晚10-11点），并要求其提交《异常情绪宣泄申请表》**
2. **将无限回廊改造为"员工健身步道"，并设置打卡点**
3. **对所有异常实体进行分类编号，建立档案管理**
4. **开庭审判违规者，依据自定的《安全区管理条例》量刑，处罚多为"强制劳动（清洁、整理）"和"规则抄写"**

这个架构将"强迫症"从一个性格癖好，升格为了一种对抗世界本质混乱的哲学武器。故事的表层是收容怪谈、经营避难所，深层则是一场关于"秩序与混乱"、"理性与疯狂"的边界战争。主角不是在求生，而是在用他偏执的秩序，为人类文明争夺最后一块"可理解"的阵地。
