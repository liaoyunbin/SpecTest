## 写作规范要求

### 每章质量标准
- [ ] 开头钩子明确（200字内）：规则出现
- [ ] 林晚反应符合人设（100字内）：吐槽漏洞、生理不适
- [ ] 规则分析透彻（300字内）：找出逻辑矛盾和格式错误
- [ ] 修正方案完美（400字内）：红笔批注，制定修正案
- [ ] 执行过程详细（500字内）：规则固化过程
- [ ] 成果展示充分（300字内）：秩序建立
- [ ] 林晚满意表达（100字内）：获得爽感
- [ ] 结尾钩子吸引（100字内）：新规则出现

### 爽点控制标准
- [ ] 每章至少一个主要爽点（规则修正）
- [ ] 每三章一个中等级爽点（区域净化）
- [ ] 每十章一个大等级爽点（势力扩张/终极对决）
- [ ] 爽点类型多样化（规则修正/秩序建立/异常收容/逻辑战争）

### 人设一致性检查
- [ ] 林晚情绪范围控制（不适→满意）
- [ ] 行为逻辑合理性（规则优先，不考虑人情）
- [ ] 标准坚持程度（规则必须规范统一）
- [ ] 理性决策体现（逻辑分析，漏洞修正）

## 战斗方式设计规范

### 面对不可直视之影
- [ ] 递上《可视光谱登记表》："请先登记你的形态和可见波长，未登记实体不予对视。"

### 面对必须回答的谜语
- [ ] 退回谜面："问题表述存在歧义，请使用标准疑问句重新提问，并提供至少两个逻辑上互斥的选项。"

### 面对杀人规则
- [ ] 添加前置义务："根据《异常实体临时管理条例》第7条第3款，你的杀人流程缺少前置告知义务，本次攻击无效。"

## 标志性台词使用规范

1. "这规则有漏洞！"
2. "格式不对，重写！"
3. "根据《管理条例》第X条第Y款..."

## 核心设定回顾

### 主角设定
- **身份**：重度规则强迫症患者，穿越到规则污染世界
- **核心能力**：【规则纠错】——看穿规则漏洞，通过红笔批注改写规则效力
- **标志性行为**：随身携带红色批注笔

### 系统设定
- **名称**：规则解析系统
- **功能**：解析规则结构、标记漏洞、辅助修正
- **神器**：无法被污染的红色批注笔、《已收容异常规则目录》

### 世界观设定
- **时间线**：污染第30天→污染第540天（约1年半）
- **地点**：丽景苑小区（被血雾笼罩的高档住宅小区）
- **规模**：10→100→500→2000人

### 三阶段架构
1. **规则管理者**（1-10万字）：整顿小区规则，建立安全区
2. **异常收容者**（10-20万字）：扩张收容，建立部门体系
3. **规则定义者**（20-25万字）：揭露真相，定义终极规则

这个任务清单为25万字小说的创作提供了清晰的执行路径，确保按时按质完成作品。
