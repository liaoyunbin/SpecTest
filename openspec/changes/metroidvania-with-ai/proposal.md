# 使用 AI 制作类银河城游戏

## 问题陈述

想用 AI 辅助做一个类银河城（Metroidvania）游戏，但目前不清楚：
- 需要掌握哪些知识领域
- 每个领域学到什么程度才够用
- AI 能在哪些环节帮忙，哪些环节需要自己来
- 从零到原型的完整实现路径

## 提议方案

系统性梳理类银河城游戏开发的完整技术体系，产出以下核心文档：

| 文档 | 用途 | 状态 |
|------|------|------|
| **CoreLoop.md** | 核心循环设计：30秒循环、三大支柱、玩家目标 | ✅ 完成 |
| **Architecture.md** | 系统架构：5层分层、20个系统、8条架构约束 | ✅ 完成 |
| **design.md** | 技术方案整合：技术选型、数据管线、场景方案 | ⏳ 待完成 |
| **tasks.md** | 实现任务：按依赖排序的开发任务 | ⏳ 待完成 |

### 核心技术选型

- **引擎**: Unity + C#
- **渲染**: 2D 横版卷轴
- **UI 框架**: UIManager (Panel + Controller 两层架构)
- **场景方案**: Persistent Scene (常驻) + Additive Scene (按需加载)
- **配置管线**: Luban (Excel → C# → 数据文件)
- **存档管线**: Newtonsoft.Json (序列化/反序列化)
- **UI 渲染**: UGUI + Canvas
- **物理**: Unity 2D Physics (Rigidbody2D + Collider2D)
- **相机**: Cinemachine

### 核心思路

- **你不必成为每个领域的专家** — AI 帮你补齐短板
- **但你需要知道"要什么"** — 才能让 AI 生成正确的东西
- **关键模块自己深刻理解** — 角色控制、关卡结构等核心体验
- **约 70% 的代码可由 AI 生成** — 重点是系统框架和重复性逻辑

## 预期成果

- 完整的游戏设计文档（核心循环 + 架构）
- 分层知识体系，明确 AI 协助边界
- 按依赖顺序的实现任务列表
- 从零到可玩原型的可行路径

## 非目标

- 不包含具体代码实现（代码在 /opsx:apply 阶段生成）
- 不涉及美术资产制作技巧
- 不比较不同引擎的优劣

## 相关约束

- 基于 Unity + C# 技术栈
- 使用 UGUI 做 UI
- 2D 横版卷轴类型
- 持久化数据使用 JSON (Newtonsoft.Json)
- 配置表使用 Luban (Excel → Luban → C#)
