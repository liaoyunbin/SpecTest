# UI 管理器（第一版）

## 问题陈述

Unity 项目中缺少统一的 UI 管理系统，导致：
- 面板打开/关闭逻辑分散，难以维护
- 面板层级混乱，没有清晰的优先级规则
- 缺乏面板缓存机制，频繁创建销毁影响性能
- 异步加载时无法处理连续点击请求
- 缺少统一的过渡动画管理

## 提议方案

开发一个单例 UIManager，提供四种面板类型的统一管理：

**四种面板类型**：
- 底层面板：应用主界面，同时只能存在一个
- 普通面板：可叠加的功能面板
- 弹窗面板：模态对话框，同时只能存在一个
- 提示面板：Toast/Loading 等，独立于主栈

**核心功能**：
- 面板栈管理：按类型规则自动处理入栈/出栈
- 缓存策略：关闭时可选缓存，避免重复加载
- 过渡动画：支持淡入淡出、缩放、滑动等
- 加载锁定：异步加载过程中忽略新请求
- 重复打开处理：按类型区分策略（忽略/置顶/刷新）

## 预期成果

- 统一的面板打开/关闭 API
- 清晰的面板层级规则
- 面板缓存和复用能力
- 流畅的过渡动画
- 稳定的异步加载处理

## 非目标

- 不实现 UI 数据绑定（MVVM）
- 不实现 UI 热更新
- 不实现复杂的 UI 布局系统

## 相关约束

- 基于 Unity UGUI 开发
- 所有产出物使用简体中文
